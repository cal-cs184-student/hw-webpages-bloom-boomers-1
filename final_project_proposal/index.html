Summary

We aim to write a real time fluid simulation built on top of the Homework 4 structure. Our goal is to also write our own rendering engine.

Problem Description

Fluid simulation is a prevalent topic in computer graphics. The accurate depiction of liquids holds value in entertainment, education, simulations, and other fields. Its implementation, however, is a challenging one. Realistically simulating fluids is computationally expensive, primarily due to the sheer number of particles and calculations that need to be accounted for to make up a beliavable fluid. Although past projects have circumvented this issue by allocating the rendering of the fluid to external rendering engines such as Blender, our project aims to implement our own rendering engine.

As previously mentioned fluid simulation computations are expensive, thus we must apply algorithms that are not only going to optimize calculations, but also do so in a GPU friendly manner since the GPU will handle most of the shading. One of the algorithms we can implement to ensure a smooth simulation is making use of a 3D grid and grouping particles by hashing them by their respective grid cell, much like what we did in part 4 of homework 4. This algorithm is explained in further detail here, but basically it allows us to cut down unnecessary calculations, while also providing the GPU with a fixed number of particles that will be used in the shading part later on.

To render liquids we will make use of the Ray Marching algorithm, more specifically Sphere Tracing, which is described here. Ray marching is a faster version of ray tracing, which will allow us to render complex reflections and refractions of the liquid in real time.

Goals and Deliverables:

Minimum Viable Deliverables (for base grade):

  • Realistic fluid physics (fluid holds itself together and bounces off of boundaries)
  • Realistic fluid rendering (including reflection and refraction of light source and environment)
  • Test scene that allows user to see reflection and refraction of environment
  • Include a webpage report and flythrough demo video.

Target Deliverables (for full grade):

  • GUI that allows user to change fluid's properties, such as color, viscosity, and density in real time
  • Fluid can interact with spheres and planes

Stretch Goals:

  • Add ocean foam to fluid
  • Interactive feature that allows user to scoop up liquid

04/14-04/20 Implement basic particle physics (particles correctly interact with boundaries and with each other). Simulation should not be to taxing for the GPU. Particles should have customizable color for debugging purposes.
04/21-04/28 Implement Ray marching rendering. Particles should clump up together and look like a fluid together (instead of a group of individual particles). Fluid doesn't need to reflect or refract light at this stage.
